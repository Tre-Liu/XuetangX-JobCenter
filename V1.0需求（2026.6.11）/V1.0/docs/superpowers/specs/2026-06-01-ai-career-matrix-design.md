# AI Career Matrix Design

## Goal

将现有“学生查看专业培养方案”的页面截图改造为更符合人工智能专业语境的高保真展示图。核心变化是重构中部“支撑矩阵”和右侧“职业生涯规划师”弹窗，使页面从原来的“毕业要求 - 指标点”关系展示，升级为面向人工智能专业的“产业 - 岗位 - 能力 - 课程”主骨架，以及“典型工作任务 / 毕业要求 / 指标点 / 知识点”辅助映射体系。

## Scope

本轮只产出单张高保真静态视觉稿，服务于汇报、演示或后续页面实现参考。

包含：

- 保留原图整体页面壳层和信息布局习惯
- 重构中部矩阵区域的信息结构与视觉关系
- 升级右侧“职业生涯规划师”弹窗，加入岗位能力图谱与课程知识点推荐
- 轻度借鉴参考素材的未来感装饰语言

不包含：

- 真实交互逻辑
- 可运行前端页面
- 动态切换动画实现
- 后端数据接口和算法推荐逻辑

## Design Summary

整体方案采用“卡片流 + 关系弧线型”。

- 中部主图以可读卡片为主，保证一眼能看懂人工智能专业的培养链路
- 多对多关系通过细弧线、节点光点和轻量层板阴影表达，不做厚重表格，也不做过度复杂的立体网络
- 右侧弹窗略微加宽，但仍从属于中部主图，避免压缩底层图谱的层级信息
- 视觉语气维持现有浅蓝白底界面，轻度引入紫蓝渐变、发光连接和悬浮感，向参考素材靠拢但不过度科幻

## Page Architecture

页面继续保持三段式结构：

1. 左侧能力导航
2. 中部培养方案主画布
3. 右侧职业生涯规划师弹窗

本次改造只重点变更第 2、3 区域。

### Left Navigation

左侧导航保持现有样式和分组节奏，不重新设计信息架构。`工程能力` 所在项可以继续作为高亮示意，但中部内容不再局限于材料工程语义，而是切换为人工智能专业的培养关系图谱。

### Center Canvas

中部画布保留：

- 顶部专业头图卡
- 页面标题与简短说明
- 培养目标达成度 / 课程完成度状态胶囊
- 下方“支撑矩阵”主展示区

其中“支撑矩阵”改造为页面核心。

### Right Drawer

右侧保留“职业生涯规划师”角色设定，但从纯问答说明面板升级为“职业建议 + 岗位能力图谱 + 课程推荐”的复合弹窗。

## Matrix Model

主图采用“4 层主骨架 + 4 层辅助映射”的结构。

### Four Primary Layers

主图横向展示四列，从左至右依次为：

1. `产业`
2. `岗位`
3. `能力`
4. `课程`

每列放置 3 至 5 张主卡片，形成稳定、易读的卡片流布局。每张卡片必须能独立阅读，不依赖鼠标悬停。

### Four Auxiliary Mapping Layers

以下四类信息不再占用完整独立列，而是作为辅助信息嵌入：

- `典型工作任务`
- `毕业要求`
- `指标点`
- `知识点`

嵌入规则如下：

- 典型工作任务：放在岗位卡片下方，作为 2 至 3 条摘要标签或小号描述
- 毕业要求：贴近能力卡片，用左侧短标签或角标形式表示
- 指标点：作为毕业要求的细分说明，放在能力卡片旁的二级文字区
- 知识点：放在课程卡片底部，以 chips 或小标签形式呈现

## Layout Specification

### Main Matrix Layout

主图不再使用现有“双列对照”形式，而改为横向四层卡片流。

布局原则：

- 每列纵向错落排布，避免机械表格感
- 列与列之间留出足够空间给关系弧线穿行
- 关系线颜色轻、宽度细，避免喧宾夺主
- 通过少量发光节点表达“多对多连接”而不是为每条关系线都加粗
- 背后可增加极淡的悬浮层板或透视底片，提示这是一张“关系矩阵”

### Card Style

卡片继续沿用现有高圆角、白底、浅描边样式，但做精致化升级：

- 轻微蓝紫渐变描边
- 更柔和的投影
- 局部高光或角落光斑
- 标题更粗，摘要更克制

不同层级可用不同强调色区分：

- 产业：浅青蓝
- 岗位：蓝紫
- 能力：湖蓝到靛蓝
- 课程：浅蓝灰

### Relationship Rendering

为了准确表达多对多，连接方式采用三层表现：

1. 主关系弧线：连接相邻主层级卡片
2. 节点锚点：在卡片边缘设置小圆点，提示关系可汇聚和发散
3. 局部发光汇点：在高频连接处放置微弱光点，增强“图谱感”

关系线只连接主层级，不额外把辅助信息拉成满屏线网，避免杂乱。

## Content Model For AI Major

本图内容采用“产教融合综合型”人工智能专业语义，覆盖算法应用、智能系统开发和工程落地三类就业走向。

### Industry Layer

- 智慧医疗
- 智能制造
- 智慧城市
- AIGC 内容服务

### Job Layer

- 机器学习工程师
- AI 应用开发工程师
- 计算机视觉工程师
- 数据智能工程师
- 智能系统运维工程师

### Capability Layer

- 数据采集与治理
- 模型训练与优化
- 视觉 / 语言智能应用
- AI 系统部署
- 工程化开发协同

### Course Layer

- Python 程序设计
- 数据结构与算法
- 数据库原理
- 机器学习
- 深度学习基础
- 计算机视觉基础
- 自然语言处理导论
- AI 系统部署实训

### Typical Task Mapping

岗位卡片下建议出现如下典型任务摘要：

- 数据清洗与标注
- 构建分类 / 检测 / 生成模型
- 开发 AI 应用接口
- 部署推理服务
- 监控模型效果并迭代优化

### Graduation Requirement Mapping

能力卡片旁的毕业要求建议选取通用工程教育表述：

- 工程知识
- 问题分析
- 设计 / 开发解决方案
- 现代工具使用
- 团队协作
- 持续学习

### Indicator Mapping

指标点文案建议具体到人工智能专业能力动作，例如：

- 能完成数据预处理与特征表达
- 能基于典型框架实现模型训练
- 能完成模型部署与效果评估
- 能在工程约束下进行应用实现与联调

### Knowledge Point Mapping

课程卡片底部知识点建议采用高频、可读、小颗粒标签：

- 监督学习
- 特征工程
- 神经网络
- 损失函数
- 模型评估
- 向量检索
- Prompt 设计
- 接口联调
- 容器部署

## Right Drawer Design

右侧“职业生涯规划师”弹窗只做有限扩宽，目标是容纳新增图谱模块，但不盖过中部主矩阵。

### Drawer Structure

弹窗内部建议分为四段：

1. 角色标题区
2. 岗位方向引导区
3. 岗位能力图谱区
4. 课程与知识点推荐区

### Guidance Area

保留原有人设口吻，文案继续以“根据当前专业选择未来就业方向”为核心，但岗位方向按钮改为更贴近人工智能专业的分组，例如：

- 算法应用型
- 智能开发型
- 数据分析型
- 平台运维型
- 产品融合型

### Capability Graph

新增“简易岗位能力图谱”，不做复杂自由网络，而采用“岗位能力节点 + 推荐连线”的轻图谱：

- 左侧是岗位核心能力节点
- 右侧是相关课程卡与知识点标签
- 中间用少量柔和连线表达推荐关系

例如：

- `模型训练` 连接 `机器学习 / 深度学习基础 / 模型评估`
- `工程实现` 连接 `Python 程序设计 / 数据结构与算法 / 接口联调`
- `系统部署` 连接 `AI 系统部署实训 / 数据库原理 / 容器部署`

### Recommendation Area

课程推荐区应更像“建议清单”，而不是第二张大图。每条建议包含：

- 课程名
- 1 句推荐理由
- 2 至 3 个知识点标签

## Visual Direction

整体视觉基调为“轻度借鉴参考样例”。

必须保留现图的可读性和产品截图感，不转成纯概念海报。建议吸收参考图的以下元素：

- 淡紫蓝环境光
- 微弱漂浮层板感
- 低饱和发光节点
- 更统一的渐变连线语言

避免：

- 大面积高饱和紫色
- 过强 3D 透视
- 节点过密导致的科技噪音
- 为了炫技牺牲文案可读性

## Data Flow And Narrative

这张图的阅读顺序应当明确：

1. 用户先在中部看到“产业如何通向岗位”
2. 再看到“岗位需要哪些能力”
3. 再看到“这些能力由哪些课程支撑”
4. 最后在右侧弹窗里看到“如果选择某一类岗位，应重点补哪些课程和知识点”

辅助信息的职责是解释“为什么连起来”，而不是再造一张独立主图。

## Guardrails

为避免视觉与信息失衡，实现时需要遵守以下约束：

- 主图最多只展示 4 个主层级
- 每列不超过 5 张核心卡片
- 连接线宁可少，不可糊成面
- 弹窗宽度只做小幅增加
- 任何新增装饰都不能遮挡文字
- 人工智能专业术语必须自然，不混入材料工程等旧内容

## Verification

验收标准如下：

1. 用户第一眼能识别这是一张人工智能专业培养链路图，而不是旧图换字
2. 中部主图能清楚读出 `产业 - 岗位 - 能力 - 课程` 四层关系
3. 画面能感受到多对多映射，但不会乱成复杂关系网
4. 右侧弹窗新增的岗位能力图谱可读、可解释推荐逻辑
5. 整体风格与参考素材存在呼应，但仍然属于现有产品界面体系
6. 画面适合用于汇报展示，不显得学术过重或技术堆砌

## Deliverable

最终交付应是一张经过重构的高保真页面图，可直接替代当前截图用于展示人工智能专业培养方案与职业发展推荐关系。
