# Decision Center Improvement Page Design

## Goal

重构 `决策中心 / 专业质量监控 / 专业改进建议` 页面，使其从当前的轻量占位页升级为一张可用于汇报演示的“看板 + 报告”型决策页面。页面需要基于实时招聘数据与行业动态，智能分析岗位需求、能力要求与人才趋势，并把这些变化明确映射到课程调整、实训增补和资源补强建议，为培养方案优化提供数据支撑。

本页需要做到三件事：

- 点明本专业对应的 `新岗位`、`新技术点` 与 `关键能力变化`
- 明确指出 `哪些课程需要调整`、`为什么调整`、`关联哪些岗位和技术`
- 给出 `建议新增的实训内容`、`资源补强项` 与 `落地节奏`

## User-Validated Direction

用户已确认以下方向：

- 页面整体风格偏 `看板` 与 `报告` 的结合，不要做成纯后台表格页，也不要做成花哨宣传页
- 顶部标题区改为 `左对齐`
- 页面采用 `双主轴并列` 的叙事方式：
  - 一条主轴描述岗位与技术趋势
  - 一条主轴描述课程与实训整改动作
- 中间关系表达最终采用 `重矩阵` 方案：
  - 第一屏保留强视觉摘要
  - 下半页用更完整、更像报告的关系矩阵讲透证据链

## Scope

本次仅重构 `专业改进建议` 单页的前端演示层与 mock 数据结构，范围包括：

- Vue 主实现中的页面结构、状态与样式
- `index.html` 静态 fallback 中的同页实现
- 本页专属 mock 数据
- 本页的默认态、刷新态、空态与数据不足态

不包含：

- 实时招聘接口接入
- 真正的行业新闻抓取
- 导出 PDF / Word
- 权限控制与审批流
- 跨系统深链跳转

## Narrative Structure

整页采用“先抓眼球，再给证据，最后落动作”的三段结构：

1. `趋势信号看板`
2. `重矩阵证据区`
3. `整改落地清单`

这样既保留决策中心应有的视觉冲击，也能把“为什么改、改什么、怎么改”讲清楚。

## Information Architecture

### 1. Header

页面顶部标题区改为左对齐，结构如下：

- 左侧：
  - 页面标题：`专业改进建议`
  - 一句方法说明：`基于实时招聘数据、行业动态与课程运行情况，形成岗位趋势、课程调整与实训补强建议。`
- 右侧：
  - `数据周期`
  - `覆盖岗位数`
  - `最近更新时间`

该区不再居中摆放大标题，避免破坏决策页的阅读起点。

### 2. Hero Signal Board

第一屏用于快速建立重点认知，采用 4 张高亮信号卡：

- `新增岗位`
- `高频新技术`
- `建议调整课程`
- `建议新增实训`

信号卡下方增加一条“本轮核心判断”摘要条，用一句可读性很强的话串起主要结论，例如：

`AIGC 应用工程师、智能体开发与多模态处理成为新增需求高点，建议优先调整人工智能导论、Python 程序设计、数据库原理，并补入提示工程与智能体工作流实训。`

这一屏负责吸睛与定调，不承载大段说明，不放复杂图表。

### 3. Evidence Matrix

页面主区采用 `重矩阵` 作为核心证据层，矩阵是整页的主报告区，而不是点缀。

矩阵至少包含六列：

- `新岗位 / 新技术`
- `岗位能力要求`
- `关联课程`
- `当前问题`
- `建议动作`
- `建议新增实训`

行级表达示例：

- `智能体开发` → `工作流编排 / 提示工程 / 工具调用` → `Python 程序设计 / 人工智能导论` → `现有课程缺少 Agent 工作流内容` → `增补章节` → `智能体工作流搭建实训`
- `AIGC 应用工程师` → `内容生成 / 数据治理 / 质量评估` → `数据库原理 / 概率论与数理统计` → `案例偏传统` → `替换案例` → `提示工程与评测实训`

矩阵目标不是铺满所有信息，而是建立清晰可信的“趋势 → 能力 → 课程 → 动作”链路。

### 4. Action Panels

矩阵下方把结论拆成三类整改清单，增强任务感：

- `课程调整`
- `实训新增`
- `资源补强`

每一条整改建议包含统一字段：

- `优先级`
- `对应岗位 / 技术来源`
- `关联课程`
- `建议动作`
- `预期收益`

这一区比矩阵更像行动板，用于帮助读者从分析过渡到执行。

### 5. Delivery Timeline

页面底部增加落地节奏分区，用于把建议转为任务安排：

- `本学期可落地`
- `下学年规划`
- `需外部资源协同`

这一区可以是三列卡片，也可以是横向阶段条，但必须能回答“先做什么、后做什么、谁来配合”。

## Visual Direction

本页延续决策中心浅蓝系风格，但信息密度高于“专业全局概览”，视觉上更偏分析报告。

视觉原则：

- 主背景保留浅蓝雾化与柔光，不额外引入新主题色系
- 重点高亮只给以下元素：
  - 新增岗位
  - 高频技术
  - 高优先级课程动作
  - 高优先级实训动作
- Hero 卡片可以比现状更有视觉冲击，但不能过度发光或做装饰型插画
- 矩阵区要更“报告化”：
  - 线框清晰
  - 行列分组明确
  - 字重克制
  - 减少视觉噪声
- 动作清单区保留看板感，但用信息层级抓眼球，不靠花哨效果

## States

本页至少需要四种状态：

### Default State

展示最近一次生成的改进建议报告，默认进入：

- Hero 信号看板
- 矩阵主区
- 整改清单
- 落地节奏

### Refreshing State

当用户触发重新分析或刷新数据时，展示：

- 顶部轻量 loading 提示
- Hero 卡片骨架
- 矩阵行骨架
- 文案：`正在同步招聘数据与行业动态，请稍候...`

不直接整页空白，不用打断式模态框。

### Empty State

当还未生成任何改进建议时，展示：

- 页面说明
- 数据来源提示
- `开始分析` 按钮
- 空态插图区或轻量装置区

空态语言要强调“基于岗位与技术变化生成专业改进建议”，而不是泛泛写“暂无数据”。

### Data Warning State

当岗位数据或课程映射不足时，展示带告警但仍可阅读的页面：

- 告警条提示缺失项，例如：
  - 岗位样本不足
  - 技术关键词覆盖不足
  - 课程映射未补全
- Hero 与已知建议仍然显示
- 缺失部分在矩阵中标明 `待补充`

不能因为部分数据缺失就把整页退回空白。

## Interaction Rules

- 左侧导航切到 `专业改进建议` 后直接进入该页默认态
- 页内可支持按以下维度筛选：
  - `岗位`
  - `技术`
  - `课程类型`
  - `优先级`
- 筛选只影响矩阵与动作清单，不重置页面整体上下文
- 支持 `重新分析`，重新进入刷新态后回到默认结果态
- 刷新浏览器后保留当前筛选条件与页面位置

## Data Model Additions

当前轻量 placeholder 数据不足以支撑该页，需要增加本页专属 mock 结构，至少包含：

- `headerMeta`
- `heroSignals`
- `headlineSummary`
- `evidenceMatrix`
- `courseAdjustments`
- `trainingAdditions`
- `resourceRecommendations`
- `deliveryTimeline`
- `warningFlags`

其中 `evidenceMatrix` 是本页主对象，不再只靠几条摘要文案完成页面。

## Implementation Notes

- Vue 主实现与静态 fallback 都要完成本页重构，避免两条链路表现不一致
- 本页不要继续复用通用 placeholder 卡片模板，需要拆出单独的渲染分支
- 样式优先复用决策中心现有变量、圆角、边框和高亮语言
- 矩阵区可使用纯 HTML/CSS 表格化布局完成，避免引入不必要图表库复杂度
- 如需增强可读性，可以在第一屏与矩阵之间插入一条简短的 `AI 摘要` 条，但不能抢主结构

## Acceptance Criteria

1. 顶部标题区左对齐，标题、说明与右侧辅助信息排布协调
2. 页面第一屏能明确看到新增岗位、新技术、课程调整和实训建议的关键摘要
3. 页面中段存在一块完整且可读的重矩阵，能清楚表达“岗位/技术 → 能力 → 课程 → 动作 → 实训”的关系
4. 页面下段存在课程调整、实训新增、资源补强三类整改清单
5. 页面底部存在“本学期可落地 / 下学年规划 / 需外部资源协同”的节奏安排
6. 页面整体仍属于决策中心视觉体系，重点突出，但不过度花哨
7. Vue 与静态 fallback 两条实现均能切换到本页并展示完整内容
8. 默认态、刷新态、空态和数据不足态均存在

## Out of Scope Follow-up

后续如果接入真实数据，可在此页继续扩展：

- 招聘来源拆分
- 技术词趋势曲线
- 岗位热度地域分布
- 一键生成课程修订任务单

这些能力不影响当前高保真前端页的交付。
