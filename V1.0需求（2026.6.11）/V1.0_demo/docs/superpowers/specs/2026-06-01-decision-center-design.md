# Decision Center Design

## Goal

从现有顶部导航的 `决策中心` 入口进入一套像素级高保真、可切换、可演示完整流程的“专业决策中心”页面群，视觉与用户提供的截图保持同一分辨率风格和布局层级，并补齐截图未覆盖但逻辑上应存在的默认态、过渡态、空态和异常提示态。

## Scope

本轮仅建设前端演示层，包含完整页面结构、状态切换、本地 mock 数据、图表与视觉装置、高保真样式和必要动效。后端接口、真实分析任务、数据持久化、权限系统、导出下载和跨页面深链分享均不在本轮范围内。

## Product Frame

决策中心挂载在现有专业详情页顶部导航中，保留当前页面壳层：

- 顶部保留专业名称、主导航、`2026版` 版本切换、`建设成果展示`、`成员`
- 左侧替换为决策中心专属能力导航
- 中间主区域承载不同分析视图

进入 `决策中心` 后，一级信息架构固定为四组：

1. `专业决策中枢`
2. `专业建设治理`
3. `专业质量监控`
4. `专业运行数据`

## Navigation Model

左侧菜单采用“分组标题 + 二级菜单”的方式组织，所有二级菜单都可点击，不保留空白入口。

### 专业决策中枢

- `专业全局概览`

### 专业建设治理

- `培养方案分析`
- `课程体系诊断`

### 专业质量监控

- `专业达成度分析`
- `学生培养分析`
- `专业改进建议`

### 专业运行数据

- `AI课程运行`
- `应用洞察诊断`
- `运行监控`

## Primary Views

### 1. 专业全局概览

这是决策中心默认落地页，对应截图中的“专业决策中枢”主视觉页面。页面中心使用分层舞台装置表达：

- 顶层：`培养目标`
- 次层：`毕业要求`
- 中层：`课程体系`
- 底层：`教学资源`

页面包含：

- 左侧虚线引导标签
- 中央发光平台与悬浮云台
- 课程标签胶囊组
- 底部资源指标卡
- 版本切换器

该页的职责是作为“总览入口”和“视觉中枢”，不承载重表格，不出现长篇分析文字。

### 2. 培养方案分析

这是 `专业建设治理` 下的重点页面组，包含完整流程，而不是单一结果页。

#### 2.1 默认待分析态

进入后优先展示方案校验与分析入口页，包含：

- 页面标题和说明
- `培养方案诊断分析 / 培养方案对比分析` 顶部切换
- 当前方案卡片
- 数据校验结果清单
- `开始分析` 主按钮

#### 2.2 分析中过渡态

点击 `开始分析` 后进入短暂的演示中过渡页，展示：

- 当前分析步骤
- 进度状态
- 骨架屏或闪烁中的卡片占位
- 简短扫描文案

该状态必须可见，不能直接瞬间跳结果页。

#### 2.3 结果态

结果页采用“报告工作台”布局，对应截图中的高保真分析报告。页面顶部保留 `重新方案分析` 和 `历史分析` 等操作位。主内容包括五个 tab：

- `综合评分`
- `数据概览`
- `培养目标智能评析`
- `毕业要求智能评析`
- `课程体系智能评析`

其中：

- `综合评分` 包含评分仪表盘、五层模型横向柱图、数据概览卡片
- `培养目标智能评析` 包含雷达图、右侧维度评分、AI分析结论、支撑强度对比图
- `毕业要求智能评析` 包含关键洞察、支撑强度分布图、预警卡、说明卡
- `课程体系智能评析` 包含课程诊断摘要、指标速览、学分结构/覆盖率等内容
- `数据概览` 作为对全量结构数据的摘要页，承接用户在总览和详细分析之间的阅读过渡

#### 2.4 空态与异常态

当 mock 数据标记为不完整时，页面需要落到异常提示版本，使用轻量告警卡提示：

- 缺少课程关联
- 缺少学分设置
- 缺少毕业要求映射
- 缺少关联教学资源

异常态仍保持完整页面骨架，不退化成空白页。

### 3. 课程体系诊断

该页面组与培养方案分析保持同一视觉系统，但内容聚焦课程结构。

#### 3.1 待诊断态

对应截图中的 `课程诊断分析` 入口页，显示：

- 页面标题与说明
- 当前方案信息卡
- 课程体系信息校验结果
- `开始分析` 按钮
- 顶部 `课程诊断分析 / 课程交叉分析` tab

#### 3.2 诊断结果态

展示课程体系的分析工作台，视觉与培养方案分析一致，但数据聚焦课程覆盖、学分分布、指标达成、问题预警和建议。

#### 3.3 课程交叉分析页

该页不是空占位，而是一个真实可切换视图，用于展示课程之间的交叉覆盖、共用支撑与潜在重复建设风险。

### 4. 其余治理、监控、运行页面

未在截图中展示的页面仍需做成可信的产品态，而不是“建设中”。这些页面统一采用同一套浅蓝决策中心视觉语言，并包含：

- 明确标题与说明
- 1 至 3 个关键指标卡
- 1 个主分析区
- 1 个 AI 结论或操作建议区

内容深度可以低于核心页面，但必须可看、可切换、可用于汇报演示。

## Interaction Rules

- 顶部点击 `决策中心` 后进入决策中心默认页 `专业全局概览`
- 左侧二级菜单切换主页面组
- 主区 tab 只切换当前页面组的内部视图，不影响左侧高亮
- 从待分析态进入分析中过渡态，再进入结果态
- 用户刷新页面后保留当前左侧菜单、当前 tab 和当前流程态
- 返回其他顶部模块后，再回到 `决策中心` 时优先恢复上次访问状态

## Visual Direction

视觉目标不是通用后台，而是严格复刻截图观感。

- 主参考分辨率为 `2048 × 1264`
- 优先保证该视口下的像素级接近，再补桌面宽度兼容
- 背景采用浅蓝底色、柔光渐变、雾化高光和几何装饰叠加
- 卡片采用半透明白底、浅蓝描边、柔和投影和大圆角
- 激活态使用蓝紫渐变、白底浮起和轻发光描边
- 中枢页的核心平台、云台、底座、椭圆阴影等装置优先用 HTML/CSS 完成
- 图表区的颜色、留白和标签风格保持与截图同一语言，不混入其他视觉体系

## Data Organization

本轮使用本地 mock 数据，按页面块拆分，而不是塞进一个总对象：

- `decisionCenterOverview`
- `planAnalysis`
- `courseDiagnosis`
- `governancePlaceholders`

每组数据同时携带：

- 页面主体内容
- tab 内容
- 默认态/过渡态/结果态/异常态所需字段

## Implementation Notes

- 延续当前工程的单页 Vue 结构，在 `src/App.vue` 中先完成一版状态驱动实现
- 新增决策中心专属状态集合，包括当前一级组、当前二级页、当前 tab、当前流程态
- 将 mock 数据和决策中心配置提取为独立常量，避免在模板中硬编码长对象
- 保持 `index.html` 直接打开时也能展示相同的决策中心流程
- 图表优先复用项目现有能力；若部分视觉装置不适合图表库，则使用 CSS/SVG 手工实现

## Verification

验收以演示质量和页面真实性为准：

1. 在 `2048 × 1264` 视口下，决策中心页面的布局比例、留白、层级和高亮关系与截图高度一致
2. 顶部 `决策中心` 入口可进入完整模块
3. 左侧菜单和主区 tab 均可真实切换
4. 截图中出现的各类页面均能访问
5. 默认态、分析中过渡态、结果态、空态和异常态均存在
6. 页面内容为真实前端结构，不以整张截图图片充当主界面
